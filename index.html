CS284A Project Proposal

Title: Simulation of water interacting with real world objects from a snapshot.

Summary: From a single snapshot using a RealSense camera, we will convert the point cloud data of objects in the scene into a geometric mesh representation. Once we have created this geometric mesh, we will perform a fluid simulation hitting the surface of those objects. Lastly, we will apply a neural style transfer of RGB images to a specific style using ML on some geometric meshes.

Members: Sela Roach, Ashwat Chidambaram, Pranav Sukumar, Riddhi Bagadiaa

  • Problem Description: The problem we are trying to solve is how fluid water interacts with different real world objects. Our project has a couple different components to it. First, we convert a point cloud of a real scene (captured by an Intel RealSense camera) to a geometric mesh that can interact with a simulated fluid. The next part would be to simulate a fluid freely falling on that object. We will need to ensure that the fluid interacts with the mesh in a realistic way.

This idea is interesting to us as it allows for an efficient method to see the interaction between water and real world objects. This simulation can extend to have real value for local government officials to test and simulate the city infrastructure in handling water during heavy rain seasons (such as the one we are going through now!). More detailed examples might include seeing how water gathers to improve flood handling methods with water flow in streets, dams, etc.

This project is challenging as it has many moving parts that must come together and work seamlessly. Such as, converting point clouds to a watertight mesh and influencing the direction of the fluid based on the shape of the mesh will be challenging to implement. We have a couple of stretch goals, the first being to add color to the geometric mesh through a neural style transfer. For instance, when we convert the point cloud to the mesh, the mesh will have no color by default, thus if we combine the RGB camera feed data with the point cloud data, we could potentially add color to the mesh, and possibly make it  more interesting by applying a neural style transfer. Another stretch goal would be to introduce this in Augmented Reality with a real-time dynamic simulation.

  • Goals and Deliverables[a]
  • Input: We will start by taking a picture of a scene using an Intel RealSense depth camera, consisting of an entire scene.
  • Output and Demo: A video of fluid interacting with the mesh. More specifically, water falls from above and hits the surface of the mesh. The mesh must be watertight such that the water doesn't seep through the surface.
  • Process:
  • Point Cloud-to-mesh - Use an Intel RealSense camera to collect dimensional data to measure objects within the snapshot, and  create a point cloud for the object. We have a few ideas for how we can implement the point cloud to mesh:
  1. We can interpolate the point cloud by adding more points until the point cloud is so dense that water particles will not seep through it. In this approach, we are using a heuristic that a dense set of points can approximate a solid surface
  2. We can implement the Ball Pivoting Algorithm for Surface Reconstruction as outlined in this paper: https://vgc.poly.edu/~csilva/papers/tvcg99.pdf 
  3. We can implement Poisson Surface Reconstruction as outlined in this paper: https://hhoppe.com/poissonrecon.pdf?from=https://research.microsoft.com/en-us/um/people/hoppe/poissonrecon.pdf&type=path 

No matter which approach we pick from above, we need to ensure that the surface is watertight and an accurate model of the real scene. A stretch goal would be to make the mesh colored by using RGB data and potentially applying a neural style transfer on top to make it look more artistic. Implementing the neural style transfer as described by: https://arxiv.org/pdf/1508.06576.pdf

  • Water simulation - After speaking with Professor O’Brien, we intend to use an off-the-shelf/existing library or package to do the particle simulation for water. A stretch goal would be to do other liquids/substances with different viscosities, or to do the simulation in real-time for a given scene.

  • Baseline Target Goal: Work with one predefined image of a scene (not real-time), and output a simulation of water falling on the object
  • Stretch Goals:
  • The user can upload an image of their choice. The program recognizes the object and makes a mesh for it. Simulate water falling on the object
  • Simulate different fluids falling on the object
  • Incorporate Augmented Reality by having a dynamic simulation of water falling on an object in real time while holding the camera up to a scene
  • Quantifiable Metrics: 
  • Percent of water retained after simulating drop over mesh (judges the quality of our mesh without any holes)
  • Time taken to render the simulation
  • Questions we plan to answer: In theory, how would water interact with different surfaces and objects present in front of us in the real world?
